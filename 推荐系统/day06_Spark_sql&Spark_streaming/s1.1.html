7.1_Spark SQL简介

7.1 Spark SQL 概述

学习目标

  • 了解Spark SQL概念
  • 了解 Spark SQL特性
  • 了解 Spark SQL优势

1 Spark SQL概念

  • Spark SQL 它是spark中用于处理结构化数据的一个模块

  • SparkSQL的前身是Shark，Shark的作用类似于hive对于mapReduce

  • Shark对于Hive的太多依赖（如采用Hive的语法解析器、查询优化器等等），制约了Spark各个组件的相互集成，所以提出了SparkSQL项目

  • SparkSQL抛弃原有Shark的代码，汲取了Shark的一些优点，如内存列存储（In-Memory Columnar Storage）、Hive兼容性等

2 Spark SQL的特性

  • Integrated: 可以在Spark程序中无缝加入SQL查询，可以使用SQL语句 也可以使用 DataFrame API. DataFrame API 提供了四种语言的实现(Java, Scala, Python and R)

    results = spark.sql("SELECT * FROM people")
names = results.map(lambda p: p.name)

  • Uniform Data Access: 使用相同的方式连接不同的数据源，包括( Hive, Avro, Parquet, ORC, JSON, 和 JDBC). 

    spark.read.json("s3n://...").registerTempTable("json")
results = spark.sql(
  """SELECT * 
     FROM people
     JOIN json ...""")

  • Hive Integration：在已有的数据仓库中执行SQL或者 HiveQL 查询，Spark SQL 支持HQL语法以及自定义函数(直接接入Hive的metaStore)

  • Standard Connectivity：提供了 JDBC or ODBC的数据接口，作为数据服务将数据通过 JDBC/ODBC方式暴露给其它BI工具

3 Spark SQL优势

  • 写更少的代码(Write Less Code)

    • RDD实现WordCount
    sc.textFile('file:///root/tmp/word.txt').flatMap(lambda x:x.split()).
map(lambda x: (x,1)).reduceByKey(lambda a,b:a+b).collect()
    • Spark SQL实现WordCount
    #直接写sql
ssc.sql('select word,count(*) as cnt from s group by word')
#通过DataFrame API
df.groupBy('word').count().show()

  • 性能更高

    • Spark SQL 提供了DataFrame API，DataFrame转换成RDD执行时自动执行代码优化，使得自动生成的RDD代码执行效率更高

      • Catalyst Optimizer：Catalyst优化器
      • ProjectTungsten：钨丝计划，为了提高RDD的效率而制定的计划
      • Code gen：代码生成器
      • 首先执行逻辑执行计划，然后转换为物理执行计划(选择成本最小的)，通过Code Generation最终生成为RDD

    • 用Java/Scala/Python编写的RDD代码比Spark SQL编写转换的RDD执行效率差

    • Language-independent API：用任何语言编写生成的RDD都一样，而使用spark-core编写的RDD，不同的语言生成不同的RDD

总结

  • Spark SQL概念：Spark SQL 与 RDD的关系 类似于 Hive 与 MapReduce的关系
  • Spark SQL 特性：Spark程序兼容性强，通用数据接口，兼容Hive，提供了JDBC和ODBC的实现
  • Spark SQL 优势：代码少，执行速度快
