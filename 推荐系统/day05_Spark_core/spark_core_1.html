6.1_Spark入门

6.1 spark 入门

学习目标：

  • 了解spark概念
  • 记忆Spark local模式启动方式

1 spark概述

  • 什么是spark

    • 专为大规模数据处理而设计的快速通用计算引擎
    • 与Hadoop的 MapReduce功能类似
    • 基于内存的分布式计算框架(只计算，不涉及存储)

  • 为什么要学习spark

    MapReduce V.S. Spark

    • MapReduce每一次shuffle都要写磁盘，多个MapReduce之间通过磁盘进行数据传递，Spark的shuffle结果可以保存在内存中的
    • MapReduce在每一个mapper和reducer之间都要经历一次shuffle，spark可以减少不必要的Shuffle
    • MapReduce的Task调度和启动开销大(进程模型)，而Spark的Task(任务)运行在线程中启动开销小
    • MapReduce编程不够灵活，仅支持Map和Reduce两种操作，Spark API更丰富

    复杂场景下Hadoop生态需要多种框架配合

    • 批处理：MapReduce、Hive、Pig
    • 流式计算：Storm
    • 交互式计算：Impala、presto
    • 机器学习: mahout

    Spark框架组件丰富

    • 批处理：Spark core、Spark SQL
    • 交互式计算：Spark SQL
    • 流式计算：Spark Streaming
    • 机器学习：Spark ML/Spark MLLib

  • spark的缺点是：内存消耗大

  • spark特点

    • 1、速度快（比mapreduce在内存中快100倍，在磁盘中快10倍）
    • 2、易用性（可以通过java/scala/python/R开发spark应用程序）
    • 3、通用性（可以使用spark sql/spark streaming/mlib/Graphx）
    • 4、兼容性（spark程序可以运行在standalone/yarn/mesos）

2 spark启动（local模式）

  • 启动pyspark

    • 在$SPARK_HOME/sbin目录下执行

      • ./pyspark

    • sc = spark.sparkContext
words = sc.textFile('file:///root/tmp/word.txt') \
            .flatMap(lambda line: line.split(" ")) \
            .map(lambda x: (x, 1)) \
            .reduceByKey(lambda a, b: a + b).collect()

    • 输出结果：

      [('python', 2), ('hadoop', 1), ('bc', 1), ('foo', 4), ('test', 2), ('bar', 2), ('quux', 2), ('abc', 2), ('ab', 1), ('you', 1), ('ac', 1), ('bec', 1), ('by', 1), ('see', 1), ('labs', 2), ('me', 1), ('welcome', 1)]

总结

  • Spark 概念
    • 基于内存的计算引擎
    • Spark比MapReduce效率更高
    • Spark组件更丰富
  • Spark的local模式启动
    • 单机模式 方便代码调试
    • WordCount案例：体会spark优势，运行速度更快，API更丰富
